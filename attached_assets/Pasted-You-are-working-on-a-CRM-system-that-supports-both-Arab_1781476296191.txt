- Change any UAE reference (AE, UAE, دولة الإمارات, +971, AED) to Egypt

## 🌐 ISSUE 2 — Arabic language is incomplete across the system

When the user selects Arabic:

### A) Full Translation Required:
- Every single page must be fully translated — no English text should
  remain visible anywhere when Arabic is selected
- All UI elements: buttons, labels, placeholders, tooltips, error messages,
  success messages, loading text, empty state messages, table headers,
  form fields, dropdown options, modal titles, notifications — ALL must
  be in Arabic
- All static data and default values displayed to the user must be in Arabic
- Section titles, page titles, breadcrumbs — all in Arabic

### B) Sidebar / Side Navigation:
- Do NOT move or flip the sidebar to the other side when Arabic is selected
- Keep the sidebar in its ORIGINAL position
- Only translate the sidebar menu items text to Arabic
- Keep icons in place, only the text labels change to Arabic

### C) RTL Layout:
- When Arabic is selected, apply RTL (right-to-left) direction to the
  content area ONLY if it doesn't break the sidebar position
- Text alignment inside cards, tables, and forms should be right-aligned
  in Arabic mode
- Do not flip the entire layout, only adjust text direction
